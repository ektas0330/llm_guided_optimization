Programming manufacturing robots with imperfect AI

LLMs as tuning experts for FDM print configuration selection

Instead of relying on an AI model to output a full print configuration, we use an LLM as a constrained decision-maker inside an evidence-driven optimization loop. A toolpath-based evaluator returns a scalar objective and structured diagnostics, which the LLM uses to propose parameter edits that steer Bayesian optimization.

Ekta U. Samani and Christopher G. Atkeson
Robotics Institute, Carnegie Mellon University

Method

A modular LLM-guided optimization loop

Given part geometry and user objectives, we cast print configuration selection as a diagnosis-driven optimization problem. Soft guidance biases selection toward configurations consistent with the proposed changes, while hard constraints restrict the next step to modifying only the implicated parameters.

1

Evaluator

Runs the slicer and returns a scalar score for optimization, feasibility vetoes, and issue-level penalties.

2

LLM guidance generator

Uses structured diagnostics and admissible actions to identify one primary issue and propose directional or categorical parameter changes.

3

LLM guidance compiler

Converts the selected actions into a soft-violation score V(x) and an implicated-parameter set I.

4

Guided optimization

Downweights candidates that contradict the guidance and freezes parameters outside I while optimizing over I.

Qualitative physical validation

Qualitative validation with physical prints

All quantitative comparisons use our evaluator because a standard scoring metric is not available. For qualitative validation, we physically print a subset of objects using each method's recommended print configuration.

Part A

Overhang curling

The default and ChatGPT configurations leave the overhang unsupported and it curls, while our method adds support beneath it to avoid the defect.

Part B

Stringing and bonding

The ChatGPT and unguided optimization configurations produce stringing between separated posts, whereas our method avoids this defect and improves bonding by choosing a lower layer height.

Part C

Delamination risk

The default configuration is flagged for delamination risk, while our method chooses a different orientation and increases perimeters to strengthen the part.
